# LGLC Website Templates - AI Assistant Rules

This file provides rules for AI coding assistants working on this project.

## Project Overview

This is a static HTML/CSS/JavaScript template project using a token-based design system for the Lesbian and Gay Liberation in Canada (LGLC) website.

## Critical Rules

### 1. Design Token System (MANDATORY)

**ALWAYS use design tokens from `css/design-tokens.css`**

✅ **DO:**
```css
.component {
  padding: var(--spacing-4);
  color: var(--color-primary-red);
  font-size: var(--font-size-lg);
}
```

❌ **DON'T:**
```css
.component {
  padding: 16px;        /* WRONG */
  color: #ED1C24;      /* WRONG */
  font-size: 18px;     /* WRONG */
}
```

### 2. CSS File Organization

- Each component gets its own CSS file in `css/` directory
- Import order is critical: `css/design-tokens.css` MUST be imported first
- Never define custom values in component CSS files
- If a new token is needed, add it to `css/design-tokens.css` first

### 3. HTML Structure

- Use semantic HTML5 elements (`<header>`, `<main>`, `<footer>`, `<nav>`, etc.)
- Include ARIA labels where needed
- No inline styles - keep all CSS in stylesheets
- Proper heading hierarchy (h1 → h2 → h3)

### 4. JavaScript

- Vanilla JavaScript only (no frameworks)
- Component-specific JS in `js/` directory
- Ensure accessibility: keyboard navigation, ARIA updates, focus management

### 5. Responsive Design

- Mobile-first approach (default styles for mobile)
- Breakpoints: `48rem` (tablet), `64rem` (desktop), `80rem` (large desktop)
- CSS variables cannot be used in `@media` queries - use values directly with comments

```css
@media (min-width: 48rem) {  /* --breakpoint-md */
  /* Tablet styles */
}
```

### 6. Accessibility

- Focus states: Yellow outline (`--color-primary-yellow`) with adequate offset
- Touch targets: Minimum `var(--touch-target-min)` (2.75rem/44px)
- All interactive elements must be keyboard accessible
- Include ARIA labels and logical tab order

## Design Token Categories

### Colors
- **Primary**: `--color-primary-red`, `--color-primary-navy`, `--color-primary-yellow`, `--color-primary-black`, `--color-primary-gray`
- **Semantic**: `--text-primary`, `--text-secondary`, `--text-tertiary`, `--text-inverse`
- **Backgrounds**: `--bg-page`, `--bg-surface`, `--bg-elevated`

### Spacing
- Use `--spacing-1` through `--spacing-24` (0.25rem to 6rem)
- Never hardcode spacing values

### Typography
- **Font sizes**: `--font-size-xs` through `--font-size-5xl`
- **Font weights**: `--font-weight-normal` through `--font-weight-extrabold`
- **Line heights**: `--line-height-tight`, `--line-height-normal`, `--line-height-relaxed`, `--line-height-loose`

### Borders & Shadows
- **Border radius**: `--border-radius-sm` through `--border-radius-2xl`
- **Shadows**: `--shadow-xs` through `--shadow-2xl`

## Component Guidelines

### Buttons
- `.btn-primary`: Main call-to-action (black background, white text)
- `.btn-secondary`: Secondary actions (transparent, black text, gray border)
- Both must have yellow focus outline

### Cards
- `.card`: Content previews, feature highlights (white background)
- `.related-card`: Related content on detail pages (icon, heading, list)
- Hover: Slight lift, enhanced shadow
- Focus: Yellow outline

### Typography
- **5xl (3rem)**: Hero titles, landing page main headings
- **4xl (2.25rem)**: Section titles, major page headings
- **3xl (1.875rem)**: Subsection headings, large content titles
- **2xl (1.5rem)**: Medium headings, section subheadings
- **xl (1.25rem)**: Small headings, card titles
- **lg (1.125rem)**: Large body text, emphasized content
- **base (1rem)**: Default body text, buttons, form inputs
- **sm (0.875rem)**: Small text, captions, metadata
- **xs (0.75rem)**: Tiny text, fine print

## Layout Patterns

### Containers
- `.landing-container`: Landing page sections (max-width: 75rem)
- `.record-container`: Individual record pages (max-width: 80rem)
- `.search-section`: Search page content (max-width: 72.5rem)

### Grids
- `.cards-grid`: 3-column grid, stacks to 1 column on mobile
- Gap: `--spacing-8`

## Common Mistakes to Avoid

1. ❌ Hardcoding CSS values instead of using tokens
2. ❌ Using `px` for spacing/fonts instead of `rem` via tokens
3. ❌ Defining custom values in component CSS files
4. ❌ Importing CSS files in wrong order
5. ❌ Using inline styles
6. ❌ Creating components without checking existing patterns
7. ❌ Not including accessibility features (focus states, ARIA)
8. ❌ Making touch targets too small (< 44px)

## When Creating New Components

1. Create new CSS file in `css/` directory
2. Reference existing tokens via `var(--token-name)`
3. If new token needed, add to `css/design-tokens.css` first
4. Follow patterns in `css/header.css` and `css/footer.css`
5. Ensure accessibility (focus states, keyboard navigation, ARIA)

## Reference Files

- `css/design-tokens.css` - All available design tokens
- `DESIGN-LANGUAGE.md` - Detailed component guidelines and usage
- `README.md` - Project structure, setup, and usage instructions
